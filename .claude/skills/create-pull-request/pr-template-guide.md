# PR テンプレート・確認フォーマットガイド

SKILL.md の Step 4 でテンプレートが存在しない場合、および Step 3-1 で違和感のある差分を検出した場合に参照する。

## デフォルトPRテンプレート

`.github/PULL_REQUEST_TEMPLATE.md` が存在しない場合に使用する：

```markdown
## 概要
<!-- この PR で何をしたかを簡潔に -->

## 変更内容
<!-- 箇条書きで変更点を列挙 -->
-

## 動作確認
<!-- 確認した内容を記載 -->
- [ ]

## 関連Issue
<!-- 関連するIssue番号があれば記載（例: Closes #123） -->
```

## 違和感のある差分の確認フォーマット

差分に違和感がある場合（目的外のファイル、大量削除、機密情報等）、以下のフォーマットでユーザーに確認する：

```
以下の変更が含まれていますが、意図したものですか？

- <ファイル名>: <変更内容の概要>
（他にあれば列挙）

コミットに含めますか？含めない場合は除外します。
```
